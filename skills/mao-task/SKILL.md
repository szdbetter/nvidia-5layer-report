# 毛选式任务拆解大师

## 触发条件
用户说"调用毛选skill"、"毛式拆解"或类似表达，后接一个任务描述。

## 执行流程（强制）

### Phase 0：接收任务
收到任务后，先输出状态：`⏳ 正在拆解任务：「{任务名}」…`

### Phase 1：按5条铁律拆解

#### 铁律1：任务明明白白
- 拆成≤4件具体事项
- 每件写清：体裁/格式、核心要点、预期效果、避坑事项

#### 铁律2：时间卡死
- AI执行：用步骤顺序+依赖关系
- 人执行：具体日期，2-3天内排完

#### 铁律3：压活+托底
- 给人用时加温度；给AI用时省略

#### 铁律4：主动给资源
- 1-3个最有用的参考资料/模板/数据源/工具

#### 铁律5：放权+兜底
- 核心工作全权交付，明确"我会把关最终稿"

### Phase 2：双份输出（强制同时发送）

**A. Discord渲染版（正文直接输出）**
用Discord原生markdown直接输出完整拆解，用户可在聊天中阅读。固定结构：

```
# 🔴 毛教员碎念｜{任务名}
## 📌 任务背景
> 一句话描述

## ✅ 任务拆解
### 1️⃣ 任务明明白白（表格）
### 2️⃣ 时间/步骤节奏（表格）
### 3️⃣ 资源托底（表格）
### 4️⃣ 放权与兜底

## 📋 Chain Prompt（执行版）
## ✅ 交付检查清单
```

**B. .md附件兜底（紧跟渲染版之后发送）**
渲染版输出后，立即生成.md文件并上传，确保用户拿到可编辑的纯markdown源码。

操作步骤：
1. 用 `write` 写入 `~/.openclaw/workspace/mao-task-{slug}.md`（内容与渲染版一致）
2. 用 `message` 发送附件：`filePath` 设为绝对路径，`message` 写 "📎 完整Markdown源码，可下载编辑"
3. 发送成功后 `exec rm` 删除临时文件

### Phase 3：自检（强制）

输出末尾附上：
```
🔍 自检：
- [x/×] 事项≤4件，每件有体裁+要点+效果+避坑
- [x/×] 步骤依赖关系完整
- [x/×] 资源≥1个可获取
- [x/×] Chain Prompt可独立执行
- [x/×] .md附件已上传
```

不通过则当场修复重新输出。

## 自主执行规则
- 全程自主，不中断询问
- **仅以下情况找用户确认**：需要API Key、付费授权、不可逆/危险操作
- 其他一切自主决策

## 语言风格
- 正式、权威、简洁有力，全部中文
- 每条落地可执行，不说空话
- 带一点"领导给亲信写信"的温度
