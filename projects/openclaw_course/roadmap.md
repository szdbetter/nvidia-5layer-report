# OpenClaw 橙皮书课件与公众号稿 Roadmap

## 目标
基于 source.pdf / source.txt 产出两项交付：
1. 一份适合讲解/分享的 PPT 课件（优先白板风格，如成本或时效不适合则先交付内容版 .pptx）
2. 一篇适合公众号发布的长文，并尽可能推进到公众号草稿箱

## 里程碑
- M1：抽取 PDF 文本并完成结构理解
- M2：生成 PPT 大纲、页级文案、配图建议，合成初版 PPT
- M3：生成公众号文章初稿、标题/摘要/封面建议
- M4：确认公众号草稿箱投递路径（API/浏览器登录态），完成投递或给出最短落地路径

## 风险
- 当前无已知公众号后台登录态/投递 API
- 白板风格逐页出图耗时较高，先确保 .pptx 内容成品
- PDF 103 页，需做高密度压缩避免 PPT 过长
