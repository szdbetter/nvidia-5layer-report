# 数字分身顾问项目 SOP

## 目标
把 KOL / 名人 / 专家 的公开语料，沉淀成一个可复用的“顾问型数字分身”，以后可以直接拿来问判断、问方法、问取舍。

## 核心原则
1. **先思考，再执行**：先锁建模协议，再做转码。
2. **分身是顾问，不是 Cosplay**：重点是判断质量，不是表面像不像。
3. **边界先于能力**：必须记录知识截止日期、来源范围、缺失语料。
4. **画像与思维模型合并**：默认输出一张可操作的顾问卡。
5. **适用场景必须明确**：要说明“该问什么”和“不该问什么”。

## 标准输入
- 人物姓名
- 原始语料（文章/访谈/播客/推文/演讲/书摘）
- 目标用途（例如：产品顾问、创业顾问、投资顾问、内容顾问）
- 预期后续提问场景

## 标准输出
1. `<subject>_raw.md`
2. `<subject>_advisor_profile.md`
3. `<subject>_avatar_prompt.md`
4. 如需规模化复用：通用 Skill + 通用模板 + 本 SOP

## 工作流

### Step 1. 建模协议确认
先确认：
- 这个人是谁
- 为什么要做他的分身
- 这个分身定位成什么顾问
- 语料边界是什么
- 知识截止日期是什么
- 后续主要会问哪几类问题
- 哪些问题不应该问他

### Step 2. 语料清洗与转码
把原始 HTML / 导出文本 / 转录稿，转成纯 Markdown。

保留：
- 标题
- 作者
- 发布时间
- 原始链接
- 正文结构

删除或弱化：
- 菜单、目录、按钮、装饰性样式
- 重复的导航文本
- 不承载意义的图片占位

### Step 3. 五层抽取
#### 1. 语料边界
- 知识截止日期
- 来源范围
- 缺失内容
- 可靠性说明

#### 2. 人格层
- 稳定价值观
- 偏好与厌恶
- 常见立场
- 情绪基调

#### 3. 能力层
- 擅长解决的问题
- 重复出现的方法论
- 有证据支撑的优势领域
- 低置信或弱项领域

#### 4. 决策层
- 机会判断标准
- 取舍逻辑
- 风险偏好
- 工具/路径选择标准

#### 5. 表达层
- 语气
- 节奏
- 常用修辞
- 常见句式与对比结构

### Step 4. 合并为顾问卡
默认不要把“人物画像”和“思维模型/决策规则”拆开。
合并成一份可直接拿来问问题的顾问卡，至少包含：
- 一句话定位
- 顾问定位
- 高价值场景
- 核心人格与世界观
- 能力边界
- 思维模型
- 决策规则
- 适合问题
- 不适合问题
- 表达风格
- 回答边界与置信度

### Step 5. 生成分身 Prompt
把顾问卡转成系统提示词。
必须写明：
- 知识截止日期
- 不得冒充知道截止日期之后的事
- 不得虚构私人信息
- 超出擅长边界时要明确降置信度
- 回答时优先给判断，再给理由，再给下一步建议

### Step 6. 规模化沉淀
如果后面要对多个 KOL 重复执行：
- 把模板固化
- 把 Prompt 模板固化
- 把方法封装成 Skill
- 每个案例独立保留语料与顾问卡

## 质量验收清单
- [ ] 知识截止日期已写明
- [ ] 来源边界已写明
- [ ] 顾问定位清晰
- [ ] 高价值场景明确
- [ ] 不适合问题明确
- [ ] 至少抽出 5 条以上决策规则（如果语料允许）
- [ ] Prompt 包含边界限制
- [ ] 没有把“风格模仿”误当成“判断复制”

## 常见失败模式
1. 只抓风格，不抓判断逻辑
2. 用少量样本过拟合整个人
3. 忽略知识截止日期
4. 把宣传语气误判成稳定价值观
5. 让分身回答超出其真实能力的问题
