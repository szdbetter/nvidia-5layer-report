# 跨频道统一数字分身顾问注册协议

## 目标
让数字分身 / 顾问 / 智囊团的创建、注册、调用，在所有频道统一生效。

## 统一触发词
以后当用户说出以下任一表达时，默认进入同一套“数字分身顾问创建协议”：

- 创建数字分身
- 生成数字分身
- 创建智囊团
- 生成智囊团
- 创建顾问
- 生成顾问

这些表达默认视为同义需求，统一按“数字分身顾问项目”流程处理。

## 默认执行流程
1. 明确对象是谁（单人 / 多人）
2. 明确用途（单顾问 / 多顾问智囊团）
3. 明确语料边界与知识截止日期
4. 收集并转为纯 Markdown 语料
5. 生成顾问卡 / 智囊团卡
6. 生成单顾问或多顾问 Prompt
7. 写入长期记忆
8. 注册到全局 registry

## 单顾问输出标准
- `<subject>_raw.md`
- `<subject>_advisor_profile.md`
- `<subject>_avatar_prompt.md`

## 多顾问 / 智囊团输出标准
- `<team>_registry.md`
- `<team>_routing_rules.md`
- `<team>_council_prompt.md`
- 每位顾问各自的 raw / profile / prompt 文件

## 跨频道生效原则
- 只要成功写入同一个 OpenViking 长期记忆库，就视为跨频道可共享
- 只要成功注册到 `projects/digital-twin-advisor/REGISTRY.md` 或团队 registry，就视为本地可检索
- 以后在其他频道中，若用户显式点名顾问名，应优先检索长期记忆和 registry，再切换到对应顾问视角

## 调用规则
### 单顾问
默认直接写人名即可，不需要额外写“分身”二字。

示例：
- `使用【刘小排】回答：<问题>`
- `刘小排怎么看这个事？`

### 多顾问 / 智囊团
用户若点名团队名、智囊团名，默认走多顾问模式：
1. 各顾问独立回答
2. 汇总一致点
3. 汇总分歧点
4. 给出综合建议和下一步动作

## 默认别名规则
- 人名 = 该顾问
- 人名 + 分身 = 同一顾问别名
- 团队名 / 智囊团名 = 多顾问集合

## 优先级
1. 用户当前消息中的显式点名
2. 长期记忆中的顾问注册信息
3. `projects/digital-twin-advisor/REGISTRY.md`
4. 各案例目录下的 profile / prompt 文件

## 统一原则
以后凡是：
- 创建数字分身
- 生成数字分身
- 创建智囊团
- 生成智囊团
- 创建顾问
- 生成顾问

都默认调用这一套统一规则，不再临时发明新流程。
