# 阶段性结论检查点

## 当前已经证明的事情

- `elon-research/` 研究链路已经可运行：`ingest -> normalize -> backtest` 能执行，测试当前为 `27 passed`。
- 跨周真实样本已经扩展到 8 周。
- 弱验证已经从“周总推文常数公允值”推进到 `proxy_tweet_windows`：
  - 接入了 `tweet_delta_5m`
  - 接入了 `tweet_delta_15m`
  - 接入了 `tweet_delta_1h`
  - 接入了 `tweet_delta_6h`
- 修复数据偏差后，原先看起来为正的跨周结果消失；当前两版跨周结果都没有给出可执行弱信号。
- 已确认 `data-api /trades` 公开接口中，当前只有 `eventId` 过滤可依赖；`slug / market / conditionId / asset` 不能作为可辩护的分段抓取方案。

## 当前还没有证明的事情

- 还没有证明存在可交易 edge。
- 还没有证明市场微结构层面的成交数据足够完整。
- 还没有拿到可公开复现的逐条历史推文时间线，因此无法进入强验证。

## 当前核心阻塞

### 1) 推文侧阻塞

- 当前只有 tracking 摘要与 hourly/daily 桶。
- 这足以做 proxy windows，不足以做逐条事件研究。
- A 级主路径仍然是 X API 历史权限。

### 2) 市场侧阻塞

- 事件级抓取只能稳定拿到每周 `3500` 条成交，因为 `offset > 3000` 会返回 400。
- market 级拆分抓取曾被验证为错误路径，已回滚。
- 抽样结果：`8/8` 高流动 market 命中该限制。
- 因此，当前市场微结构研究仍需保守解释，不能把成交表当作严格完整历史。

## 当前最合理的研究判断

- 现阶段证据更接近“尚未观察到可辩护 edge”，而不是“已经找到可交易 edge”。
- 若继续投入，最有价值的工作不是继续调弱验证参数，而是先解决两个数据主阻塞：
  - 逐条历史推文时间线
  - 单 market 成交完整性

## 建议的下一阶段顺序

1. 优先推动 X API 历史权限或等价 A 级来源。
2. 寻找替代成交来源或可分段抓取路径，绕开单 market `offset > 3000` 限制。
3. 只有在上述两项至少解决一项后，再进入下一轮策略筛选。
