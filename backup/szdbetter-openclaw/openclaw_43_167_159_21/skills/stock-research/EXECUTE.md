# 调研执行指南

## 场景切换规则

### 美股场景（默认）
触发条件：
- 用户明确要求美股
- 或未指定市场，且标的/赛道默认按美股处理

使用文件：
- `phase1-prompt.md`
- `phase2-prompt.md`

快速启动：
```bash
qmd run us-research-phase1 --sector "AI Agent"
```

---

### 港股场景（AI Agent / 机器人 / 模型基础设施专用）
触发条件：
- 用户明确要求港股
- 且赛道属于：AI Agent / 大模型 / 机器人 / 推理云 / AI基础设施 / Agent工具链 / AI安全

使用文件：
- `phase1-hk-agent-prompt.md`
- `phase2-prompt.md`（二阶段框架通用，但执行时补充港股特有字段）

执行要求：
1. 不得覆盖美股版流程
2. 必须区分：✅直接受益 / 🔄映射受益 / 🎭情绪龙头
3. 必须补充港股字段：港股通 / 配售减持 / 解禁 / 流动性 / 做空风险
4. 不得把“涨得猛”直接当 Alpha

---

## 通用执行流程

### 1. 识别市场
- 美股 → 读取 `phase1-prompt.md`
- 港股 AI Agent 场景 → 读取 `phase1-hk-agent-prompt.md`

### 2. Step 0 候选池生成
- 美股：按美股版 prompt 生成候选池
- 港股：按港股版 prompt 生成候选池，并按 5 层分类：
  - 模型层
  - Agent应用层
  - 推理/云基础设施层
  - 具身智能层
  - 数据/安全/工具链层

### 3. Step 1+2 数据抓取
- 优先抓实时行情、营收、核心业务趋势
- 若数据源受限，必须标 `[⚠️未验证]`，但仍输出可执行结论

### 4. Step 3 催化剂与风险扫描
- 提炼每家公司最近 1 条最重要催化剂
- 同时写出 1 条最关键风险
- 港股额外检查：配售/减持/解禁/港股通/做空

### 5. 评级与结论
- 美股：按原 4 分制
- 港股：按港股版 5 分制
- 输出最终结论时，必须直接回答：
  - 哪里有 Alpha
  - 哪里只是情绪

### 6. 输出位置
- Phase 1：`workspace/reports/phase1/[日期]_[赛道].md`
- Phase 2：`workspace/reports/phase2/[日期]_[TICKER].md`

## 硬规则
1. 先识别市场，再选 prompt
2. 港股版不能覆盖美股版
3. 无验证数据必须显式标 `[⚠️未验证]`
4. 必须说人话，不能只堆术语
5. 结论必须能支持后续二阶段深挖决策
