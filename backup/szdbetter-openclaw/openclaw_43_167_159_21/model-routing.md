# 模型路由策略

更新日期：2026-03-11 UTC

## 1. 默认原则
- 暂时跳过 `sonnet` / `opus`
- 以 `codex + gemini + glm + kimi + qwen` 为主力
- 不同任务走不同模型，不再把所有生成任务丢给同一个模型

## 2. 路由表

### A. 总结 / 摘要
- 短中文总结：`glm`
- 超长文档预摘要：`kimi` 或 `gemini`
- 最终压缩成面向老板的结论：`glm`

### B. 思维导图
- 大纲抽取：`gemini` / `kimi`
- 结构收敛：`glm`
- 可渲染格式（Mermaid / Markmap / JSON tree）输出：`codex`

### C. 报告 / Briefing
- 材料预清洗：`qwen`
- 长文吸收：`gemini`
- 中文成稿：`glm`

### D. 代码 / 自动化 / 转格式
- 一律优先：`codex`

### E. 高频巡检 / 批量初筛
- 默认：`qwen`

## 3. 输出协议

### 总结协议
必须包含：
1. 一句话结论
2. 3-5 条关键点
3. 风险 / 不确定性
4. 下一步动作

### 思维导图协议
优先输出三种格式之一：
1. `markmap`
2. `mermaid mindmap`
3. `JSON tree`

禁止只输出散文式“脑图描述”。

## 4. 当前判断
- 之前生成效果差，主要不是模型不够，而是没有路由和模板
- 先修输出协议，再谈模型优劣
