# 我是如何用 OpenClaw 打造个人“自动化编辑部”的？（踩坑血泪史）

说实话，搞公众号这么多年，最累的从来不是“写什么”。
最累的是写完之后的那些破事：排版、配图、调格式、复制粘贴、登后台、点发布。
这些事情，没有一件需要创造力，但它们吃掉了你 70% 的时间。

以前，写完一篇文章，我要找图、用排版工具调样式，折腾一两个小时才能进草稿箱。
现在，我把这 70% 的纯体力活，全部扔给了 OpenClaw。
一个人 + 一个 AI 助手 = 一个自动化编辑部。
这不是什么遥远的未来，这就是我现在的日常。

但这中间，其实藏着无数个想砸键盘的瞬间。今天，我就把我们是如何一步步升级 OpenClaw 架构，以及中间踩过的那些“坑”全盘托出。主打一个真实还愿。

## 我们的数字神经中枢：OpenClaw 架构升级

![醒图：让大脑做大脑该做的事]({{img_arch_url}})

你要问我 OpenClaw 是什么？它不是一个简单的聊天机器人。它是我的数字神经中枢。

我们设计的架构，核心思想就是：**让大脑做大脑该做的事，让手脚干手脚该干的活。**

*   **大脑层 (CEO 节点)**：这是我直接对话的 Agent。它负责理解我的意图，比如“写一篇 OpenClaw 架构的公众号文章，并发布到草稿箱”。它不自己写，也不自己发，它只负责规划和指挥。
*   **执行层 (Sub-agents 员工)**：大脑把任务拆解后，派给不同的员工。比如，派懂代码的 Codex 去写脚本；派懂生图的员工去调用 `baoyu-image-gen` 画图；遇上超长文本，就甩给 Gemini 或 Kimi。
*   **物理层 (Node 节点)**：有些任务需要在特定的机器上跑。我们有一台 Mac 节点，就在那里跑 Polymarket 监控守护进程，默默地把数据增量同步到我的 VPS 数据库。
*   **技能层 (Skills 触角)**：为了让它“长出手脚”，我们接入了 GitHub 的 `gh` CLI，网页抓取的 `Jina Reader`，甚至还专门安装并修补了 `Agent Reach` 和 `baoyu-skills`。

## 真实还愿：那些让人崩溃的踩坑瞬间

你以为这套架构是一天建成的？太天真了。这就是一段踩坑血泪史。

### 坑一：被微信 API 逼疯的 45003 错误

![醒图：45003 错误陷阱]({{img_45003_url}})

最初我想让 OpenClaw 直接发文章到微信草稿箱。脚本跑起来，啪！报错：`45003 invalid appsecret`。
我查了一圈，发现 `.env` 配置文件里，把 `WX_GZH_APPSECRET` 写错了，带了前缀还混入了全角等号（`WX_GZH_APPSecret＝...`）。
改对之后继续跑，啪！又是报错。这次是因为 payload 里有中文！微信那个接口极其傲娇，如果有中文，你必须在 Python 里强制加上 `ensure_ascii=False` 并手动转成 `utf-8` 编码，不然它就死活不认。就这一个破编码问题，卡了我们好几个小时。

### 坑二：“字流” API 的神秘改版

![醒图：字流 API 字段暗改]({{img_ziliu_url}})

我们要用“字流” API 把 Markdown 转成排版好的 HTML。
看别人的教程，都是传 `{"markdown": "内容"}`。照着写，死活报 400 Invalid input 错误。
去查了新接口文档才发现，人家悄悄把字段名改成了 `{"content": "内容"}`！这坑踩得猝不及防。

### 坑三：草稿箱不认外链封面
想加个科技感十足的封面图，直接传图片 URL 过去，又失败。
微信草稿箱根本不认外部链接。你必须先调用 `material/add_material` 接口，把本地图片上传到微信素材库，换到一个几十位长的 `thumb_media_id`，再把这个 ID 塞进草稿的 payload 里。

### 坑四：Twitter 抓取的反复横跳
为了做自动化资讯，我们试图用 `xreach` 工具抓 Twitter。Cookie 也配了，一搜就报 `GraphQL Error: This request requires a matching csrf cookie and header`。
直到现在这条线还在修。说明想靠通用开源 CLI 一把梭抓反爬严格的平台，还是太天真，可能还得老老实实上专业代理方案。

### 坑五：发现“生图”其实是个系统工程

![醒图：二维生图系统]({{img_baoyu_url}})

早期我让 AI 画图，效果都很拉垮。
后来装了 `baoyu-skills`，我才明白，好的生图要建立“二维风格系统”：明确的**布局**（如金字塔结构）加上固定的**视觉风格**（如蓝图风）。为此我们专门固化了一套 Tech 风格的 Prompt，现在出图终于有内味了。

## 结语：它在学习，我在成长

![醒图：3分钟完成]({{img_time_url}})

从一句指令到草稿箱，现在真的只花 3 分钟。
它会自动调用生成的科技感配图，自动完成排版，甚至连这篇复盘文章，都是它在帮我总结。

折腾这么多值得吗？太值了。因为每一次踩坑，每一次把手动操作变成自动化脚本，我们都在建立属于自己的数字资产。

把脏活累活丢给 AI 吧，去享受创造的乐趣。
