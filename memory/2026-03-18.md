## LCM vs OpenViking 架构分析（2026-03-18）
- LCM（Lossless Context Management）占contextEngine插件槽
- OpenViking占memory插件槽
- 两个槽位独立，不冲突，可并行
- LCM = 会话级上下文管理（压缩但不丢原文，DAG分层摘要，SQLite存储）
- OpenViking = 跨会话知识记忆（提炼后的知识点，向量数据库语义检索）
- 当前OpenClaw版本2026.3.13，支持contextEngine槽
- 安装命令：openclaw plugins install @martian-engineering/lossless-claw
- 装完后两系统互补：LCM管会话内不失忆，OpenViking管跨会话知识沉淀
