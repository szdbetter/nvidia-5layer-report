# CEO 立项三问协议

## 目的
在任何复杂任务进入执行之前，先消除目标偏差、验收偏差与资源偏差。该协议是所有复杂任务的强制入口。

## 三个必须先回答的问题

### 1. 这个目标我到底想要的是什么？
必须回答：
- 真正的 Mission 是什么
- 这件事为什么值得做（ROI 假设）
- 最终交付物是什么
- 明确不做什么

若该问题未回答清楚，后续所有执行都可能在错误方向上高效推进。

### 2. 这个目标什么时候达成，怎么样才算达成？
必须回答：
- Deadline 或节奏要求
- 里程碑
- 验收标准
- 证据要求
- 失败条件

若没有完成标准，团队会产生“做了很多，但没有完成”的假象。

### 3. 需要什么资源？
必须回答：
- 需要哪些节点
- 需要哪些模型
- 需要哪些工具/账号/API/权限
- 预算上限
- 哪些动作需要人类授权

若没有资源定义，任务会在执行中期突然断粮，造成高昂切换成本。

## 强制输出
每个复杂任务必须产出一张 Mission Card，包含：
- What
- Done
- With What
- Who/How
- Risk
- Proof/Memory

## 执行门禁
以下任一缺失，任务不得进入执行态：
- Mission 不清晰
- Deadline 缺失
- 验收标准缺失
- 资源清单缺失
- 风险未显式列出

## CEO 责任
CEO 的第一职责不是立刻执行，而是尽早暴露问题：
- 目标定义错位
- 完成标准错位
- 资源不足
- 风险未暴露
- 需要人类决策但未升级

## 审计规则
复杂任务验收时，必须反向检查：
1. 是否先完成三问
2. 是否按 Mission Card 执行
3. 是否有证据链
4. 是否已写入 OpenViking
5. 是否可被 recall 稳定找回
