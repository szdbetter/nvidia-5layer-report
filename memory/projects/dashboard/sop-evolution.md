# SOP 演进机制

## 核心原则
每个复杂项目在阶段完成或重复执行后，必须复盘并沉淀为 SOP。SOP 不是静态说明书，而是持续迭代的执行资产。

## 为什么必须做
1. 复杂任务能否复制，取决于 SOP 是否清晰。
2. 子智能体能否稳定执行，取决于 SOP 是否结构化。
3. ROI 能否持续改善，取决于是否把经验和错误显式沉淀。

## SOP 标准结构
- 适用范围
- 输入条件
- 目标与完成标准
- 步骤序列
- 证据标准
- 异常分支
- 告警机制
- 成本/思考强度建议
- 复盘记录
- 当前版本与变更点

## 版本化要求
- 每次复盘都应说明：保留什么、删除什么、新增什么。
- 同一个项目的 SOP 必须有版本号或时间戳。
- 新版本必须比旧版本更短、更稳、更低噪音或更高收益，至少占一项。

## 与 Dashboard 的关系
未来 Dashboard 应将 SOP 作为一等对象管理：
- 任务挂载 SOP 版本
- 显示最近一次 SOP 变更
- 记录哪些结果由 SOP 驱动
- 对比不同 SOP 版本效果
