# Decision

## 2026-03-09
### DECISION-001 输出边界
- 决策：Polymarket 项目相关汇报、战报、执行结果只发当前线程 `# › Polymarket-🇺🇸🇮🇷美伊战争`，除非用户明确要求发往其他频道。
- 原因：用户已明确要求“消息发在这个Channel，不要其他频道了，除非我要求”。
- 优势：信息集中、便于回溯、避免串频道污染与证据分散，同时保留用户临时指定外发的控制权。
- 证据：频道消息 `1480574159506378874`、`1480588461793738832`。

### DECISION-002 单市场 MVP 策略
- 决策：只围绕 `US x Iran ceasefire by March 31? -> NO` 跑单市场闭环，不并行扩到多市场。
- 原因：符合 AGENTS.md 的“单市场 MVP 原则”，先验证信号、规则、复盘，再扩张。
- 优势：降低噪音、降低误判、复盘清晰、ROI 可量化。
- 证据：频道战报与持仓记录，含 `1480514203725987872`、`1480554126986317965`。

### DECISION-003 统一作战引擎策略
- 决策：使用统一引擎，把盘口检查与 OSINT 检查合并到同一流程，而不是拆成两个独立系统。
- 原因：用户明确要求“一个进程，30分钟检查一次靖安 和 poly价格”；价格与情报割裂会错过信息差套利窗口。
- 优势：避免盘面与情报脱节，减少错误止损/漏报，形成同频决策面。
- 证据：频道消息 `1480508368287432809`、`1480508687364915341`、`1480508688543514665`。

### DECISION-004 事件市场方法论
- 决策：交易“概率变化”和“市场反应偏差”，不交易新闻标题本身。
- 原因：事件市场利润来自信息尚未完全计价的时间差，而非新闻复述。
- 优势：可将 OSINT 前置信号转化为领先于盘面的概率判断，形成信息优势。
- 证据：MEMORY 中 Polymarket OSINT 记录；频道内统一引擎与 OSINT 战报演化。

### DECISION-005 OSINT 六维信号框架
- 决策：采用靖安文章拆出的六维信号框架做基础模型：战略空运、侦察活动、高层飞行、航母/通信破绽、基地卫星/加油机、六维加权评分。
- 原因：用户要求“我想要的是这里的数据源，你是取哪里的”；必须可解释、可追溯。
- 优势：信号源是物理世界前置变量，早于新闻标题与市场价格；可复用到自动化监控。
- 证据：频道消息 `1480502195282509856`、`1480502739606962347`、`1480502740747816983`、`1480502742098120887`。

### DECISION-006 报告字段策略
- 决策：战报必须逐步包含价格、Polygon 地址、成本价、盈亏、OSINT 来源/方法。
- 原因：用户明确要求“战报增加显示，Polygon地址，成本价，盈亏，以及我怎么知道你取OSINT是怎么来的”。
- 优势：提高可审计性，用户可直接判断系统是否在真实运行。
- 证据：频道消息 `1480546728032604353`、`1480554126986317965`。

### DECISION-007 风险授权边界
- 决策：未获明确授权前，只执行只读查询、监控、验证；真实下单/approve/资金划转必须进入 cowork 再执行。
- 原因：涉及资金与策略切换，属于高风险操作。
- 优势：控制资产风险，保留用户最终决策权。
- 证据：项目协作规范 + 本项目 cowork 记录。

### DECISION-008 Polymarket 执行拓扑迁移
- 决策：所有 Polymarket 策略运行面迁移到 Fiona 节点，巡检频率调整为分钟级；异常统一回流给 Reese。
- 原因：用户明确要求 Reese 负责大脑职能，执行、监控、日常运行由节点承接。
- 优势：主线程聚焦策略、开发、分发与异常处置；执行面与决策面分层，便于扩展多节点运行。
- 证据：本线程用户指令（2026-03-10 12:36 GMT+8）。
