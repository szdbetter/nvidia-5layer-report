## 为什么会报“找不到私钥”
我刚核对了 `scripts/poly_unified_engine.py` 的实际逻辑，问题在这里：

```python
with open("/Users/ai/.openclaw/workspace/config/.secrets") as f:
    for line in f:
        if 'PRIVATE_KEY=' in line:
            pk = line.strip().split('=', 1)[1].strip('"')
            break
```

这段代码有三个硬伤：

1. **没优先读环境变量**
   - 它不是先 `os.environ.get("PRIVATE_KEY")`
   - 而是直接硬编码去读 `config/.secrets`

2. **路径硬编码**
   - 直接写死 `/Users/ai/.openclaw/workspace/config/.secrets`
   - 当前运行环境如果不是这个路径，或者文件挂载方式不同，就会失败

3. **异常被吞了**
   - 外层是：
   ```python
   try:
       ...
   except: 
       pass
   ```
   - 所以真实错误不会暴露出来
   - 可能是文件不存在、权限问题、路径不对、内容格式不匹配，但最后都被伪装成“找不到私钥”

所以结论不是“配置没有”，而是：

> **脚本读取私钥的方式错误，且把真实异常吞掉了，导致误报成“找不到私钥”。**
