## 二、核心方法论提炼

### 2.1 规则模糊性识别四步法

```
Step 1: 定位量化缺失
  → 搜索"严重"、"广泛"、"多数"等无具体数字的形容词
  → 例：伊朗霍尔木兹"严重限制"无百分比定义

Step 2: 检测逻辑矛盾
  → 规则前后条款是否存在互斥
  → 例：格陵兰"US AND Denmark" vs "executive order alone"

Step 3: 识别单点依赖
  → 是否过度依赖单一信源（AP、特定政府机构）
  → 该信源失败时规则如何处理？

Step 4: 外部事件覆盖测试
  → 选举无效、会议取消、职位合并等边缘场景
  → 规则是否明确覆盖？
```

### 2.2 收益评估公式

```
预期收益 = 市场流动性 × 规则歧义概率 × 可操作窗口(小时)

高价值机会特征:
  - 流动性 > $20M
  - 歧义概率 > 30%（双方各有合理依据）
  - 可操作窗口 > 2小时（澄清发布前）
```

### 2.3 可复用模式库

| 模式 | 适用场景 | 关键信号 |
|------|---------|---------|
| 信源语言精确性 | 所有"官方声明"类盘口 | 检查是否指定exact wording |
| 结算机制失效 | 事件结果可能被取消/延期 | 规则是否提及annulment场景 |
| 数值舍入歧义 | 利率、价格阈值类盘口 | 非标准变动的舍入方向 |
| 跨市场传导 | 同主题多盘口 | 一事件触发多市场 |
