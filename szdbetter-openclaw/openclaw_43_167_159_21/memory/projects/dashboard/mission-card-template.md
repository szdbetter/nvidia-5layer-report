## 7. 证据与沉淀（Proof / Memory）
- 产物路径:
- 日志/截图/样本:
- 关联 session/process/cron:
- OpenViking 写入状态:
- Recall 验证结果:

## 使用规则
1. 复杂任务未填写完 Mission Card，不进入执行态。
2. 无 Deadline、无验收标准、无资源清单，禁止派单。
3. 无证据的“完成”视为未完成。
4. 核心里程碑完成后，必须写入 OpenViking 并做 recall 验证。
