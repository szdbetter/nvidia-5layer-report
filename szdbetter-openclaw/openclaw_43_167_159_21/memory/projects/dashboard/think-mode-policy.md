# Think 模式强度策略

## 目标
在质量与 token 成本之间做动态平衡，不把高强度思考浪费在低复杂任务上。

## 默认规则
### 低强度 / 关闭 think
适用于：
- 单步查询
- 固定 SOP 执行
- 低风险格式整理
- 明确输入输出的机械任务

### 中强度 think
适用于：
- 多步骤但路径清晰的任务
- 需要小范围取舍
- 轻量架构修改
- 多源信息汇总

### 高强度 think
适用于：
- 高偏差成本任务
- 架构设计
- 模糊目标澄清
- 多约束冲突
- 战略判断 / ROI 判断
- 一旦做错代价很大的任务

## 复盘要求
每次复杂任务结束后，记录：
- 本次任务复杂度判断是否准确
- think 强度是否过高或过低
- 是否有 token 浪费
- 下次同类任务建议强度

## 长期目标
形成“任务类型 -> 推荐 think 强度”的经验映射，并作为 SOP 的一部分长期维护。
