## 1. xtracker API 结构

**xtracker.polymarket.com** 是 Polymarket 官方的社交账号推文追踪追踪器后端。

### 1.1 API 端点清单（已验证）

| 端点 | 方法 | 说明 | 响应格式 |
|------|------|------|---------|
| `/api/users` | GET | 所有被追踪用户列表 | `{success: true, data: [User...]}` |
| `/api/users/<handle>` | GET | 单个用户详情（含 trackings 列表） | `{success: true, data: User}` |
| `/api/users/<handle>/posts?platform=X` | GET | 用户所有推文原始记录 | `{success: true, data: [Post...]}` |
| `/api/trackings?activeOnly=false` | GET | 所有追踪任务（含 elon 的共 115 个） | `{success: true, data: [Tracking...]}` |
| `/api/trackings/<id>?includeStats=true` | GET | 单个追踪任务的统计数据（每日明细） | `{success: true, data: {stats: {...}}}` |
| `/api/metrics/<userId>?type=daily` | GET | 用户每日汇总指标 | `{success: true, data: [Metric...]}` |

## 4. X Cookie 方案技术参数

**数据来源：** xtracker.polymarket.com API 分析 + 已知技术文档

### 4.1 xtracker 数据获取方式推断

从 xtracker API 响应结构推断：
- `platformId: "44196397"` → 使用 Twitter 官方用户 ID（Elon 的 Twitter ID 是固定的 44196397）
- `platform: "X"` → 明确区分 X vs other platforms
- `importedAt` 时间戳比 `createdAt` 晚几分钟 → 批量导入机制
- `platformId` 格式为 Twitter snowflake ID（如 `2034983617341137406`）

**推断的技术栈：**
1. 使用 Twitter/X cookies 或 API 获取用户时间线
2. 使用 `GET /2/users/:id/tweets` 端点（需要 Bearer Token）
3. 解析 snowflake ID 提取时间戳
4. 存储到 xtracker 数据库

### 4.2 Cookie 方案关键参数

```
User-Agent: Mozilla/5.0 (需要真实浏览器 UA)
Cookie 必需字段:
  - auth_token (或 guest_token)
  - ct0 (CSRF token)
  - twid (用户 ID cookie)
  
Headers:
  Authorization: Bearer <BEARER_TOKEN>
  x-csrf-token: <CSRF_TOKEN>
  
端点:
  GET https://x.com/i/api/graphql/<query_id>/UserTweets
  或
  GET https://api.twitter.com/2/users/:id/tweets
```

### 5.4 Polymarket 盘口分区

xtracker 追踪发现，Elon 推文盘口存在以下时间段粒度：
- **月度盘口**：30-32天（如 2025-11 全月=923条，日均29.8）
- **周度盘口**：7-8天（最常见，日均约40-70）
