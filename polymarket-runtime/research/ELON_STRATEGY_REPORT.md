## 第一章：机会本质（First Principles）

### 1.1 为什么这个盘口存在 Alpha？

**大多数参与者的定价方式：**
- 凭直觉/近期印象估算（"Elon最近发挺多的"）
- 看 xtracker 页面的实时数字做线性外推
- 极少有人建立量化模型或做条件概率分析

**我们的信息不对称：**

| 维度 | 市场参与者 | 我们 | 优势类型 |
|------|-----------|------|---------|
| 历史数据 | 看几周 | 141天完整每日数据 | 精度优势 |
| 预测模型 | 线性外推 | 条件概率分布（Day5→Day7） | 精度优势 |
| 实时数据 | xtracker页面（手动刷新）| API轮询+Cookie方案 | 时间优势（5分钟） |
| 规则理解 | 粗略了解 | 精确解析计数规则边界 | 精度优势 |

**核心判断**：这是一个**精度优势为主、时间优势为辅**的机会。大多数参与者不会花时间建立统计模型，他们的定价误差在周期中期（Day 3-5）最大。

### 1.2 Alpha 的规模上限

**关键结构性优势：feesEnabled = false（零手续费！）**

这是极其罕见的。Polymarket 绝大多数盘口收取 2% 的手续费，但 Elon 推文盘口免手续费。这意味着：
- 即使只有 2-3% 的边际优势也能盈利
- 高频小额交易可行（无手续费摩擦）
- 套利空间完全归交易者所有

**每次交易的期望收益估算：**

假设我们在 Day 6（已知 6/7 数据）入场，此时预测正确率 ≈ 85%（后续推导）：

```
场景：YES 价格 = $0.60，我们预测概率 p = 0.85
期望收益 = p × (1 - price) - (1-p) × price
         = 0.85 × 0.40 - 0.15 × 0.60
         = 0.34 - 0.09 = $0.25/share (25% edge)
```

**流动性约束**：
- 周成交量 $17M，日均 $2.4M
- 最后 24 小时成交量预估占 40%（约 $960K）
- 保守单次建仓上限：$50K-$120K（不超过流动性 5%）

**单次交易最大期望收益**：$50K × 25% = **$12,500**（保守场景）

### 1.3 Alpha 衰减风险

**市场抗性评估：**
- 当前活跃盘口成交量 $17M/周 → 市场参与者众多但多为散户
- xtracker API 完全公开 → 技术壁垒极低，但统计建模壁垒中等
- 盘口已运行数月 → 若有高频量化团队介入，边际会被压缩

**可持续性预估：**
- 短期（1-3个月）：边际充裕，竞争者少 → **高置信度盈利**
- 中期（3-6个月）：可能出现1-2个模仿者 → **边际压缩至10-15%**
- 长期（6个月+）：需要持续优化模型 + 扩展到其他人物盘口

## 第三章：规则精确解析

### 3.1 计数规则原文解读

> "only main feed posts, quote posts and reposts will count. Replies will NOT count — however, replies on the main feed such as https://x.com/elonmusk/status/1786073478711353576 will be counted by the tracker."

**拆解：**

**计入的推文类型：**
1. **Main feed posts（原创推文）** — 毫无歧义
2. **Quote posts（引用推文）** — 毫无歧义
3. **Reposts（转发）** — 即 RT，毫无歧义
4. **Main feed replies** — 这是灰色地带

**"Main feed replies" 的触发条件：**
X/Twitter 的机制：当用户回复自己的推文时，该回复会出现在 main feed（时间线）上。此外，当用户回复另一个大 V 的推文且该回复获得足够互动，X 算法可能将其提升到 main feed。

从 xtracker 的 `posts` API 抽样观察，大量 `RT @...` 格式内容被记录。xtracker 的判断标准是：**如果推文出现在用户的公开时间线页面上，就计入**。

**估计 main feed replies 占比**：约 5-10% 的推文。这是**双向风险**——xtracker 可能计入我们漏算的，或漏算我们看到的。

### 3.2 删除推文的影响

> "Deleted posts will count as long as they remain available long enough to be captured by the tracker (~5 minutes)."

**Elon 删推文频率：**
- 已知行为：Elon 偶尔删推文（争议性推文、打字错误等）
- 估计频率：约 1-3% 的推文在发布后 24 小时内被删除
- 大部分删除发生在发布后 1-2 小时内

**关键**：由于 xtracker 每 5 分钟抓取一次，绝大多数推文在被删除前已被捕获。只有**发布后 5 分钟内删除**的推文才会丢失——这种情况极为罕见（估计 < 0.5%）。

**实际影响**：可忽略不计。删除的推文几乎都会被 xtracker 记录。

### 3.3 Community Reposts

> "Community reposts which are not counted by the tracker not count toward the total."

这指的是 X Communities 功能中的"社区转发"。当 Elon 在 Community 中转发内容时，这类转发不出现在 main feed 上，xtracker 不会抓取，因此不计入。

**影响**：Elon 极少使用 Communities 功能（他主要在 main feed 发推），此类情况 < 1%。

### 3.4 xtracker 与直接计数的预期偏差

| 偏差来源 | 方向 | 幅度 |
|---------|------|------|
| Main feed replies 被 xtracker 计入但我们可能漏算 | xtracker 偏高 | +2-5 条/周 |
| 5分钟内删除的推文 xtracker 漏计 | xtracker 偏低 | -0-1 条/周 |
| Community reposts xtracker 不计 | 一致 | 0 |
| **净偏差** | **xtracker 略偏高** | **+1-4 条/周** |

**策略含义**：由于结算以 xtracker 为准，我们的预测必须对标 xtracker 的计数逻辑。最安全的做法是**直接用 xtracker API 数据做预测**，而非自己数推文。Cookie 方案仅用于最后时刻的领先优势。

## 第四章：技术实现方案

### 方案 A：xtracker API 直接对接（推荐主方案）

**优点**：
- 数据与结算完全一致（xtracker 就是结算数据源）
- 免费、无需认证、无限制
- 实现简单，维护成本为零

**缺点**：
- 5分钟延迟（仅在最后时刻有影响）

### 方案 B：X Cookie 实时监控（辅助方案）

**适用场景**：周期最后 2 小时，当前累计在档位边界附近

**实现要点**：
1. 从浏览器 DevTools 提取 `auth_token` 和 `ct0` cookies
2. 使用 X 内部 GraphQL API `UserTweets` endpoint
3. 轮询间隔：30秒
4. Cookie 有效期：7-30天，需定期刷新

### 方案 C：组合策略（最终推荐）

```
常规（Day 1-6）: xtracker API 每 3 分钟轮询
关键期（Day 7 最后 4 小时）: xtracker API 每 1 分钟 + Cookie 每 30 秒
冲刺期（最后 15 分钟）: Cookie 每 5 秒
```

完整实现见 `elon_tracker.py`。

### 5.2 Kelly 仓位计算

**半 Kelly 公式：**

```
f* = (p × b - q) / b
half_kelly = f* / 2

其中：
  p = 预测该档位 YES 的真实概率
  q = 1 - p
  b = (1 - price) / price  （YES 价格对应的赔率）
```

**实例计算：**

Day 6 结束，累计 280 条，预测 Day 7 总计落在 320-339 区间：

```
历史条件概率分析：
  Day6=280 → Day7 = 280 + (48.9 ± 25.1) = 329 ± 25
  P(320-339) ≈ 30%  （正态分布近似）
  P(300-319) ≈ 25%
  P(340-359) ≈ 22%

假设 320-339 档 YES 价格 = $0.18
  b = 0.82/0.18 = 4.56
  f* = (0.30 × 4.56 - 0.70) / 4.56 = (1.37 - 0.70) / 4.56 = 0.147
  half_kelly = 7.3%

如果总资本 = $50K → 建仓 $3,650
预期收益 = $3,650 × (0.30 × 0.82 - 0.70 × 0.18) = $3,650 × 0.120 = $438
```

### 5.3 最佳入场策略：Day 7 最后 4 小时

**为什么最后 4 小时是黄金窗口？**

- 已知 6.83/7 天的数据 → 仅需预测最后 4 小时
- 4 小时预期推文数 ≈ 11.5 ± 8（基于每小时 2.9 条）
- 当前累计 + 11.5 ± 8 → 95% CI 跨约 1-2 个档位
- 此时正确档位的 YES 价格通常在 0.50-0.80

```
场景：最后 4 小时，累计 = 310，预测最终 = 321.5 ± 8
  P(320-339) ≈ 60-70%
  如果 YES 价格 = $0.55
  b = 0.45/0.55 = 0.818
  Edge = 0.65 × 0.45 - 0.35 × 0.55 = 0.293 - 0.193 = 0.100 (10% edge)
  half_kelly = 6.1% → $3,050（$50K base）
  预期收益 = $305
```

**但如果在更极端的边际情况：**
```
最后 1 小时，累计 = 335，预测最终 = 337.9 ± 2.9
  P(320-339) ≈ 75%
  如果 YES 价格 = $0.65（市场尚未完全定价）
  b = 0.35/0.65 = 0.538
  Edge = 0.75 × 0.35 - 0.25 × 0.65 = 0.263 - 0.163 = 0.100
  half_kelly = 9.3% → $4,650
  预期收益 = $465
```

### 5.4 negRisk 结构的利用

negRisk 结构意味着每个档位是独立的 YES/NO 市场，且所有档位的 YES 概率之和 = 1。

**套利机会**：如果所有档位 YES 价格之和 > 1.0，存在无风险套利（买入全部 NO）。但由于 feesEnabled=false，这种错误定价可能更频繁出现。

**实操**：
- 监控所有档位 YES 价格之和
- 如果 Σ(YES prices) > 1.02，买入全部 NO → 无风险收益 2%
- 如果 Σ(YES prices) < 0.98，买入全部 YES → 无风险收益 2%

## 附录 A：关键数学公式汇总

### A.1 条件概率预测

给定 Day k 累计推文数 $C_k$，预测 Day 7 最终总数 $T$：

$$T \sim C_k + (7-k) \times \mu_{daily} + \sqrt{7-k} \times \sigma_{daily} \times Z$$

其中：
- $\mu_{daily} = 48.9$（历史日均）
- $\sigma_{daily} = 25.1$（历史日标准差）
- $Z \sim N(0,1)$

$$P(T \in [a, b]) = \Phi\left(\frac{b - C_k - (7-k)\mu}{\sqrt{7-k}\sigma}\right) - \Phi\left(\frac{a - C_k - (7-k)\mu}{\sqrt{7-k}\sigma}\right)$$

### A.2 Kelly Criterion

$$f^* = \frac{p(b+1) - 1}{b}, \quad f_{half} = \frac{f^*}{2}$$

### A.3 期望收益

$$E[R] = p \times (1 - price) - (1-p) \times price = p - price$$

当 $p > price$ 时存在正期望。

### A.4 最大回撤估算

$$MDD_{n} = 1 - (1 - f_{half})^n$$

连续 n 次满亏的概率：$(1-p)^n$
